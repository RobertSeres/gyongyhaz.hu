## 2025-05-14 - [Mobile Navigation & Modal Accessibility]
 **Learning:** Mobile navigation and modal interactions often lack synchronized ARIA states and visual feedback. Using a unified state management function for the mobile menu ensures that `aria-expanded`, `aria-label`, and icon classes (`fa-bars` vs `fa-xmark`) remain consistent regardless of how the menu is toggled (e.g., button click vs navigation link). For popups, distinguishing between full-screen modals (`aria-modal="true"`) and sticky non-modal prompts (using `aria-hidden` management) is crucial for proper focus management and accessibility tree visibility.
 **Action:** Always use a unified toggle function for multi-trigger UI components and apply the correct combination of `role="dialog"`, `aria-modal`, and `aria-hidden` based on whether the component should trap focus or remain a non-blocking part of the page.
